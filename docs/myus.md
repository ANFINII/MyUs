# MyUs - 総合メディアプラットフォーム SNS 設計書

## サービス概要

MyUsは「My Idea Know Us」を由来とする、YouTube型UIを採用した総合メディアプラットフォームSNSです。動画、音楽、漫画、画像、ブログ、チャットなど多様なメディアコンテンツの投稿・共有が可能で、リアルタイム通信機能も備えた統合型プラットフォームです。

## 主要機能

### コンテンツ投稿機能
- **Video**: 動画アップロード
- **Music**: 音楽アップロード
- **Blog**: リッチテキストエディタ対応ブログ
- **Comic**: 漫画投稿
- **Picture**: 画像投稿
- **Chat**: リアルタイムチャット

### ユーザー機能
- 認証システム
- プロフィール管理
- マイページ機能
- 通知設定
- 料金プラン管理

### インタラクション機能
- いいね機能
- コメント・リプライ
- フォロー/フォロワー機能
- 検索機能
- 通知システム

## データモデル詳細

### ユーザーシステム

#### User
- 認証情報
- アバター画像
- アクティブ状態、スタッフ権限
- 関連モデルへの自動生成トリガー

#### Profile
- 個人情報
- 連絡先
- 住所情報
- 自己紹介文

#### MyPage
- 公開プロフィール情報
- バナー画像
- フォロワー/フォロー数カウント
- 広告表示設定
- Google Tag Manager ID

#### UserNotification
- メディアタイプ別通知設定
- アクション別通知設定

#### UserPlan
- 料金プラン情報
- Stripe顧客ID、サブスクリプションID
- 支払い状態
- プラン開始/終了日

### チャンネルシステム

#### Channel
YouTubeのチャンネルと同様の概念。1人のユーザーが複数のチャンネルを所有でき、各チャンネルにメディアコンテンツを投稿する。

- **owner**: チャンネル所有者
- **ulid**: ユニーク識別子
- **avatar**: チャンネルアバター画像
- **name**: チャンネル名
- **description**: チャンネル説明
- **is_default**: デフォルトチャンネルフラグ
  - ユーザー作成時に1つのデフォルトチャンネルが自動生成される
  - メディア投稿時、明示的な指定がない場合はデフォルトチャンネルに投稿
- **created/updated**: 作成/更新日時

#### チャンネルの利用フロー
1. ユーザー登録時にデフォルトチャンネルが自動作成される
2. メディア投稿時、ユーザーのデフォルトチャンネル（`is_default=True`）を取得
3. メディアはチャンネルに紐づいて作成される
4. メディア一覧・詳細では `channel.owner` から作者情報を取得

### メディアコンテンツ

#### 共通機能（MediaModel）
- **channel**: 所属チャンネル
- **ulid**: ユニーク識別子
- タイトル、説明文
- カテゴリー、ハッシュタグ
- いいね機能
- 閲覧数カウント
- 公開/非公開設定
- 作成/更新日時

#### Video（動画）
- 動画ファイル、変換済みファイル
- サムネイル画像
- 検索対応

#### Music（音楽）
- 音楽ファイル
- 歌詞テキスト
- ダウンロード許可設定
- 検索対応

#### Comic（漫画）
- メイン画像
- ComicPageによる複数ページ管理
- ページ順序制御

#### Picture（画像）
- 画像ファイル
- シンプルな画像投稿機能

#### Blog（ブログ）
- リッチテキスト
- プレーンテキスト
- サムネイル画像

#### Chat（チャット）
- 有効期限設定
- リアルタイム通信対応
- スレッド数、参加者数カウント
- スコア計算機能

### インタラクションシステム

#### Comment
- 階層構造
- メディアタイプとIDによる汎用的な関連付け
- いいね機能
- リプライ数カウント

#### Message
- Chat内でのメッセージ
- スレッド対応
- TipTapエディタ対応

#### Follow
- フォロワー/フォロー関係管理
- 双方向の関係を記録

#### Notification
- 送信者/受信者
- 通知タイプとオブジェクトの汎用的な関連付け
- 確認済み/削除済み状態管理

### マスターデータ・その他

#### Plan
- 料金プラン定義
- Stripe API ID
- 価格、広告表示上限

#### Category / HashTag
- 日本語/英語名称管理
- メディアコンテンツとの多対多関連

#### SearchTag
- ユーザー毎の検索タグ管理
- 表示順序制御

#### AccessLog
- IPアドレス記録
- アクセス対象の記録

#### Advertise
- 広告コンテンツ管理
- 画像/動画対応
- 表示期間設定
- 全体/個別配信設定

## 技術アーキテクチャ

### バックエンド
- **言語**: Python 3.14
- **フレームワーク**: Django 6.0
- **API**: django-ninja
- **ASGI サーバ**: Daphne
- **WebSocket**: Django Channels + channels-redis
- **データベース**: MySQL
- **認証**: JWT + カスタム認証バックエンド
- **依存管理**: uv
- **静的解析**: mypy + ruff

### バックエンドのレイヤー構成（`myus/api/src/`）
ドメイン駆動の 3 層構造で構成。

- **domain/**
  - `interface/`: dataclass による DTO と repository interface
  - `entity/`: Django ORM Model
- **usecase/**: ビジネスロジック層
- **adapter/**: HTTP エンドポイント
- **types/schema/**: API の入出力スキーマ
- **routers.py**: ルータの集約・登録
- **injectors/**: DI ヘルパ

### フロントエンド
- **フレームワーク**: Next.js 16
- **言語**: TypeScript 5.9
- **ライブラリ**: React 19
- **HTTP**: axios + neverthrow
- **エディタ**: TipTap
- **動画**: video.js + HLS
- **国際化**: next-i18next
- **依存管理**: pnpm
- **静的解析**: ESLint + Prettier
- **テスト**: Jest + Testing Library

### リアルタイム通信
- **WebSocket**: Django Channels + Redis
- **対象**: Chat メッセージ、スレッド機能

### 外部連携
- **決済**: Stripe API
- **メール**: SMTP
- **動画変換**: ffmpeg

### パフォーマンス最適化
- **インデックス**: 主要検索フィールドに設定
- **Select/Prefetch Related**: N+1 問題対策
- **bulk_get / bulk_save**: 一括取得・保存パターンを repository interface に統一
